Browser Events : 
    1. event : events are just announcement (e.g. click, double click, keypress, scroll, up key, down key)
               There is action attached with every event like if we click on submmit button then submit the particular 
               Events are nothings just announcements done by the Browser
               To check events, we need to type monitorEvents(document); and to uncheck it we need to type unmonitorEvents(document);

    2. event-target : it is a entity where the event is recieved and possibly we have used event-listener and event handler
                      event-target example : if there is a clickable button on webpage then that button is event-target (we can call elements of html an event-target)

    3. event-listener :  a. It is a action perform after the click, scroll, key press and that action is written on specific location and that
                         b. location is event listener (agar event-listener ke andar ye define karte he ke event-target pr ye target aaye to kya 
                            action perform karni he)
                         c. to add event listener = addEventListener()
                         d. to remove event listener = removeEventListener()
                         e. syntax = 
                            where,
                            (event-target).addEventListener((event-type), (function => action));
                            (event-type) = kis event ki baat ho rhi he
                            (function => action) = ye function jo action darshata ho ki ye action aane pe kya karna he
                         f. Inshort, clicking on button is an event and button is event-target then action perform after the click is event-listener

    4. event-handler : function that perform action is the event-handler


    5. Phase of event : It is a life cycle of an event, the way in which event is traversing within DOM heirarchy
                    <div>
                        <article>
                            <h1></h1>
                            <p></p>
                        </article>
                    </div>
                    DOM tree : div
                                |
                             article
                                |
                            ---------
                            |       |
                           h1       p
        lets say if we clicked on <p></p> when how will it traverse, we will see
        1. capturing phase : click => div => article => p (this is capturing Phase, it is way by which we reach to event-target).
        2. at target Phase : p
        3. bubbling phase : p => article => div (it traverse throught its parent element till which we reach to root element).
        Whenever we use .addEventListener() in JavaScript, by default the event is handled in the bubbling phase.
        if we want to change this in capturing phase then we need to pass 'useCapture' value as true or false in .addEventListener
        e.g. addEventListener(type, event-listener, useCapture); 

    6 Event Object : Whenever an event trigger then that event goes to .addEventListener as a parameter and we can utilise it to see which type of 
                     data is stored in the event (The parameter (event) that gives details about the triggered event)
                     example : 
                                     function changeText(event) {
                                         console.log(event);  // this will show a event on browser console
                                         let fpara = document.getElementById("fpara");

                                     }
                     to check event-object in browser, we have to perform event on browser then on console we will get an event then dragdown that event then under that dragdown there will be type mention of the event

    7. default action : It actually stops tag to executes its operation and push its own operation (operations like links, forms, checkboxes)
                        Example : if there's aan anchor tag that will redirect to some link then if we add "event.preventDefault();" in listener 
                                  and we pushed in click type of event then that link wont be open from the browser and we can even define our own action inside listener
