JavaScript Runtime Environment :
    JavaScript is interpreted, single threaded and synchronous by default.
    (JavaScript by default ek time pe ek hi kaam karti hai aur line by line code execute karti hai)

    1. JavaScript Runtime Environment (JRE) is a container which has everything required to run JavaScript code.
       (JRE ek complete environment hai jisme JS code run karne ke liye sari required cheeze hoti hain)

       JavaScript Runtime Environment (JRE) is a complete environment required to run JavaScript code.
       (JRE ek complete environment hai jisme JS code run karne ke liye sari required cheeze hoti hain)

       It contains JS Engine, Web APIs, Event Loop, Callback Queue and Microtask Queue.
       (Isme JS Engine, Web APIs, Event Loop, Callback Queue aur Microtask Queue hote hain)

    2. JRE contains :
            • JS Engine
            • Web APIs
            • Event Loop
            • Callback Queue
            • Microtask Queue

       (Ye sab milke JavaScript ko properly run karte hain)

    3. JS Engine is the heart of the JRE.
       (JS Engine actual component hai jo JavaScript code execute karta hai)

    4. Every browser has its own JRE to run JavaScript code.
       (Har browser ke paas apna runtime environment hota hai JS run karne ke liye)

    5. To run JavaScript outside the browser we use Node.js.
       (Browser ke bahar JS run karne ke liye Node.js use hota hai)

    6. Some APIs are common in browser and Node.js like :
            • setTimeout()
            • setInterval()

       But internally they work differently.
       (Dono environments me APIs same lag sakti hain but internally implementation alag hoti hai)

A. JS Engine Architecture :
    JS Engine converts JavaScript code into machine code and executes it.
    (JS Engine JS code ko machine code me convert karke execute karta hai)

    JS Engine contains :
            • Call Stack
            • Heap Memory

    1. Call Stack :
            Used for function execution and execution context management.
            (Functions execute karne aur execution contexts manage karne ke liye use hota hai)

    2. Heap Memory :
            Used for storing objects and dynamic memory allocation.
            (Objects aur dynamic data store karne ke liye use hota hai)

    JS Engine Internal Flow :

        |JS FILE|
            ↓
        |PARSER|
            ↓
        |AST|
            ↓
        |CONVERSION TO MACHINE CODE|
            ↓
        |MACHINE CODE|
            ↓
        |CODE EXECUTION|


        1. JS File :
                Human readable JavaScript code.
                (Ye normal JS code hota hai jo humans likhte hain)

        2. Parser :
                Checks code for syntax errors.
                (Ye syntax errors check karta hai)

                If syntax error found then execution stops.
                (Agar syntax error mila toh code run nahi hota)

        3. AST (Abstract Syntax Tree) :
                Parser converts code into a tree-like structure.
                (Parser code ko ek tree structure me convert karta hai)

        4. Conversion To Machine Code :
                JS Engine converts AST into machine code.
                (JS Engine AST ko machine code me convert karta hai)

        5. Machine Code :
                CPU understandable code.
                (Ye CPU ko samajhne wala code hota hai)

        6. Code Execution :
                Machine code executes using Call Stack and Execution Context.
                (Execution Call Stack aur Execution Context ki help se hota hai)



    Execution Context :
        Environment where JavaScript code executes.
        (Ye wo environment hai jaha JS code execute hota hai)

        Types :
                • Global Execution Context
                • Function Execution Context



    Execution Context contains :

        1. Memory Creation Phase :
                Memory allocation happens before execution.
                (Execution start hone se pehle variables/functions ko memory milti hai)

        2. Code Execution Phase :
                Actual code executes line by line.
                (Actual code line by line execute hota hai)

        3. Lexical Environment :
                Stores current scope information and outer scope reference.
                (Ye current scope aur parent scope ki information rakhta hai)

                Lexical Environment contains :

                    • Environment Record
                        Stores variables, functions and parameters.
                        (Variables, functions aur parameters ko store karta hai)

                    • Outer Environment Reference
                        Reference to parent lexical environment.
                        (Parent scope ki taraf point karta hai)

                        Used to create scope chain.
                        (Scope chain banane ke liye use hota hai)



B. Web APIs :
    Browser/Node provided asynchronous features.
    (Browser ya Node ke extra async features)

    Examples :
            • setTimeout()
            • fetch()
            • DOM Events
            • addEventListener()


C. Callback Queue :
    Stores completed asynchronous callbacks.
    (Completed async callbacks temporarily yaha wait karte hain)


D. Microtask Queue :
    Stores high priority asynchronous tasks.
    (High priority async tasks yaha store hote hain)

    Contains :
            • Promise.then()
            • Promise.catch()
            • Promise.finally()
            • queueMicrotask()

    Microtasks execute before callback queue tasks.
    (Microtasks callback queue se pehle execute hote hain)



E. Event Loop :
    Continuously checks whether Call Stack is empty or not.
    (Ye continuously check karta hai stack empty hai ya nahi)

    If stack becomes empty :
            • First executes all microtasks
            • Then executes callback queue tasks

    (Pehle microtasks execute hote hain fir normal callbacks execute hote hain)

    Priority :
            Microtask Queue > Callback Queue
