                                                             AJAX

        (AJAX (fetch() ya fir axios) ki help se browser background me server se request bhejta hai aur response lekar page ko update karta hai, bina page refresh kiye.)
    1. Ajax is a Asynchronous Javascript and XML(eXtensible Markup Language).
    2. XML is format of data which is deprecated now
    3. We will learn how to make HTTP Requests to server in backend (HTTP is the way to communicate with the server)
    4. Ajax is a set of web development techniques using many web technologies on "client-side" to create asynchronous web applications.
    5. AJAX is an asynchronous technique that asynchronously communicates with the server to fetch data without interrupting the current execution or reloading the page and it can be implemented using XMLHttpRequest, Fetch API, or libraries like Axios.
    6. We dont use XML format anymore, instead modern way is to use JSON now.
    7. We have 3 most common ways to create and send request to server
        1. XMLHttpRequest (Old way)
           - Uses the XHR API
           - Used in traditional AJAX implementations
           - it returns => Raw response (usually string)
           - To convt this string to json, use => JSON.parse(xhr.responseText)
        2. Fetch API (Modern way)
           - Uses fetch()
           - Promise-based API provided by the browser
        3. Axios (Third-party library)
           - Uses XMLHttpRequest in the browser
           - Uses http/https module in Node.js

    8. Data fetching flow :
        🧑‍💻 (User)   (Initiates the action (like opening the page or clicking a button).)
                |
                v
        💻 [HTML + JS Page]     (HTML provides the structure (buttons, list, etc.)JavaScript handles logic (events, AJAX, DOM updates).)
                |
                v
        🖱️ Click -> 🔁 AJAX (fetch/axios)   (JS listens for the click (addEventListener). On click, JS starts an asynchronous operation → AJAX (fetch() or axios))
                |
                v
        📡 HTTP Request (POST/GET/etc.) Every API comes with methods which are mentioned in its documentation
            (
                JS sends HTTP request to server.
                    GET → get data (browser by default make GET request)
                    POST → send new data
                    PUT → update data
                    PATCH → update part of data
                    DELETE → remove data
            )
                |
                v
        🌐 Server (CRUD + DB)
            (
                Reads the request
                    Applies CRUD operation (Create, Read, Update, Delete)
                    Sends back data (usually in JSON format)
            )
                |
                v
        📩 JSON Response    (JS receives data asynchronously (without page reload))
                |
                v
        🧠 JS updates HTML (DOM)    (JS takes that data and dynamically updates the web page)
